sung.base

Base functionalities for the sung package.

This module provides foundational tools and abstractions for interacting with Spotify’s API. It includes classes and functions to search for tracks, manage playlists, and handle Spotify track metadata. The utilities here are designed to integrate seamlessly with Spotify’s API and streamline data extraction and manipulation.

Key Components: - search_tracks: A function to perform searches on Spotify, supporting filters such

as year, genre, and market.

  • TracksBase: A base class for managing collections of Spotify tracks, providing a dictionary-like interface with additional utilities for metadata handling.

  • Tracks: A concrete implementation of TracksBase that uses track IDs or track metadata.

  • PlaylistReader and Playlist: Classes for managing Spotify playlists, offering read-only and mutable interfaces respectively.

  • Utility functions and constants for managing Spotify clients and data extraction.

This module is foundational for building higher-level operations within the sung package.
